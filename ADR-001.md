# ADR-001: Premium Multi-Provider Strategy (Hugging Face + Groq)

## Context
The AI Book Factory has evolved from a "zero-cost" utility into a **Premium Automated Publishing Engine**. To compete in the professional KDP market, it requires top-tier reasoning (Qwen 2.5 72B / Llama 3.1 70B) and elite visual generation (FLUX.1-dev).

While Groq remains the fastest for secondary tasks, the primary goal is now **Highest Output Quality** over simple zero-cost constraints, while still leveraging available free-tier/low-cost API credits where possible.

## Decision
We have implemented a **Tiered Multi-Provider Architecture**:

1. **Primary Text Engine (Premium)**: **Hugging Face Inference API** utilizing `Qwen/Qwen2.5-72B-Instruct` and `meta-llama/Llama-3.1-70B-Instruct`. These provide superior prose depth and technical accuracy.
2. **Primary Visual Engine (Elite)**: **FLUX.1-dev** on Hugging Face. This model produces professional publication-grade covers with genre-specific archetypes.
3. **High-Speed Fallback (Groq)**: Used as a failover for text generation if HF capacity is reached or for rapid metadata processing.
4. **Resiliency Fallback (Schnell/Pillow)**: Automatic visual degradation to `FLUX.1-schnell` or Local Pillow renders to ensure 100% uptime.

## Consequences

- **Quality Dominance**: The prose quality has jumped from "good AI" to "professional technical author" level.
- **Visual Superiority**: Covers are now indistinguishable from high-end photorealistic or artistic designs, moving away from "AI-looking" hallucinations.
- **Provider Resilience**: The system is no longer dependent on a single API. If Groq is rate-limited or HF is at capacity, the system cross-rotates providers automatically.
- **Async Efficiency**: The implementation of `asyncio.to_thread` ensures that heavy API calls (especially 45s+ FLUX.1-dev renders) do not block the system's task loop.
- **Configuration Overhead**: Users now manage `LLM_PROVIDER` and `IMAGE_MODEL` toggles in `config.py`, providing flexibility for different cost/quality ratios.
